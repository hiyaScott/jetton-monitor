# 跨平台构建环境需求说明

> 如果你想知道：我需要什么条件才能帮你构建 Windows 和 macOS 版本？

---

## 现状

**当前环境：** Linux (Ubuntu)

**能构建：** ✅ Linux 版本 (.deb, .AppImage)

**不能构建：** ❌ Windows (.exe, .msi) 和 macOS (.dmg, .app)

---

## 为什么 Linux 无法直接构建 Windows/Mac？

### 技术原因

| 平台 | 需要 | 当前环境 |
|------|------|----------|
| **Windows** | MSVC 编译器 + Windows SDK | ❌ 无 |
| **macOS** | Xcode + macOS SDK + 签名证书 | ❌ 无 |

简单说：这些系统需要自己的专用工具链，Linux 上无法直接生成。

---

## 方案对比：如何获得 Windows/Mac 构建能力

### 方案 A：云构建服务（GitHub Actions）⭐ 推荐

**已配置完成**

| 优点 | 缺点 |
|------|------|
| 全自动，无需人工干预 | 每次构建需等待 10-20 分钟 |
| 同时构建所有平台 | 需要网络访问 GitHub |
| 免费 | 大型文件下载可能超时 |

**你需要的：** 只需等待 GitHub Actions 完成

---

### 方案 B：租用云服务器

如果我需要直接构建 Windows/Mac 版本，需要：

#### Windows 云服务器
- **平台：** AWS/Azure/阿里云 Windows Server
- **配置：** 4核8GB+，100GB SSD
- **费用：** 约 ¥2-5/小时
- **时间：** 首次环境配置 2-3 小时，后续构建 20 分钟/次

#### macOS 云服务器
- **平台：** AWS EC2 Mac / MacStadium
- **配置：** 8核16GB+（Apple Silicon 或 Intel）
- **费用：** 约 ¥10-20/小时（macOS 云主机很贵）
- **时间：** 首次环境配置 2-3 小时

**成本估算（构建一次）：**
- Windows：¥5-10
- macOS：¥30-50
- 合计：¥35-60/次

### 方案 C：本地虚拟机（复杂，不推荐）

在 Linux 上运行 Windows/macOS 虚拟机：

| 平台 | 可行性 | 难度 |
|------|--------|------|
| Windows | ✅ 可行 | 中等 |
| macOS | ⚠️ 灰色地带 | 高 |

**问题：**
- macOS 虚拟机违反 Apple EULA（使用协议）
- 性能很差，构建极慢
- 配置复杂

## 推荐方案总结

| 场景 | 推荐方案 | 原因 |
|------|----------|------|
| 长期使用 | GitHub Actions | 免费、自动、可靠 |
| 紧急一次 | 你运行一键脚本 | 最快、最省事 |
| 批量构建 | 租用 Windows 云服务器 | 成本可控 |
| macOS 构建 | GitHub Actions | macOS 云主机太贵 |

---

## 结论

对于"小白"用户，**最简单的方式**是：

1. **Windows 用户** → 下载 `build-windows.bat`，右键管理员运行
2. **macOS 用户** → 下载 `build-macos.sh`，终端运行
3. **不想折腾** → 等待 GitHub Actions 自动构建

其他方案要么成本高（云服务器），要么技术复杂（交叉编译），要么有法律风险（macOS 虚拟机）。
